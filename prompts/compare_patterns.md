# Compare Design Patterns

Compare how different products handle the same design challenge.

## Steps:
1. Use `compare_design_approaches` with the target page type
2. For each example, note:
   - Layout strategy and structure
   - Key UI elements used
   - Behavioral patterns (loading, empty, error states)
   - Visual style and color approach
   - Accessibility considerations
3. Identify commonalities (these are user expectations)
4. Identify unique approaches (potential differentiators)
5. Recommend the strongest approach with reasoning

## Output Format:
- **Common patterns**: What most examples share
- **Unique approaches**: Interesting differentiators
- **Recommendation**: Best approach for the user's context
- **Blueprint**: Synthesized design specification
